 - 삼중 Nested Loop로 해결... 했는데..
    - 문제 자체가 시간복잡도가 중점이 아닌, **모든 경우의 수를 계산**해보는 것이 중점!

 - 무조건 빠른 알고리즘 보다는 **문제 조건에 따라 코딩을 빠르게 할 수 있는 알고리즘**을 생각하는 것이 중요!
